# Resume — Design Guidelines

## Design Context

### Users
Developers who switch between multiple AI coding assistants (Codex, Claude, Amp, Pi) throughout the day and need to find or resume past sessions. The app is opened with a goal in mind — find the session about X, copy its resume command, get back to work — and closed quickly. Users are technical, keyboard-heavy, and have high taste in developer tooling. They reach for this app the way they'd reach for Spotlight or Raycast: it is a reference utility, not a destination.

**Job to be done:** Locate a specific past AI coding session by fragments of memory (folder, prompt, file, tool, recency) and resume it with one keystroke.

### Brand Personality
**Confident · Professional · Precise.**

The interface should feel like premium developer tooling — earning trust through density, typographic precision, and restraint. No marketing voice, no friendly mascots, no hand-holding empty states. Copy is terse, technical, and accurate. Numbers are tabular. Paths are monospace. Status is a small pill, not a banner.

### Aesthetic Direction
**Linear-style cool monochrome with a single restrained indigo accent.**

- Surfaces: near-white in light, deep blue-black in dark. Hairline 1px borders, no decorative shadows.
- Type: Geist + Geist Mono. Tight letter-spacing. Small uppercase eyebrows for section headers.
- Density: medium-high. Result rows ~58px. Generous use of monospace for paths, stats, command snippets, and metadata.
- Accent: `#5e6ad2` / `#8a8ff7` (light/dark) — used sparingly for active state, focus rings, primary actions, and brand mark only.
- Motion: subtle and functional. 80–120ms hover transitions, a spinning refresh icon, and `prefers-reduced-motion` respected. No perpetual loops, no decorative animation.

**Reference apps that capture the right feel:** Linear, Raycast, Vercel dashboard, GitHub's command palette.

**Anti-references (must NOT feel like):**
- Generic Material/Bootstrap admin templates — boxy cards, default shadows, blue links.
- AI-slop aesthetics — purple gradients, neon glows, glassmorphism without purpose, oversized hero text.
- Chunky consumer apps (Slack, Notion) — big buttons, large paddings, friendly rounded corners.

### Theme
Both light and dark, following the system via CSS `light-dark()`. Both modes are first-class — no half-finished light theme.

### Accessibility
WCAG AA floor. Contrast ratios meet AA for text and interactive states. `prefers-reduced-motion` is honored (already wired). Keyboard navigation is primary: `⌘K` focuses search, arrow keys move selection, `⌘C` copies the resume command, and focus rings are visible (accent-tinted, 3px). Touch targets are not optimized for — this is a Mac desktop app.

### Design Principles

1. **Density earns trust.** Resist padding. Prefer hairlines and negative space over cards and shadows. If a box doesn't communicate elevation or grouping, remove it.
2. **One accent, used surgically.** Indigo signals "active," "focused," "primary." Never decorative, never as a gradient, never for branding flourish. Everything else lives in cool neutrals.
3. **Monospace is a signal, not a style.** Use Geist Mono for anything the user might want to copy, compare, or count: paths, commands, stats, timestamps, source tags. Body copy stays sans.
4. **Keyboard-first, mouse-tolerated.** Every action has a shortcut, every shortcut is discoverable via a `kbd` element or tooltip. Hover states are bonuses; focus states are the spec.
5. **Quiet by default.** Status badges are pills, errors are inline, refresh is a thin strip. No modals, no toasts, no celebration. The app's job is to disappear after the resume command is copied.
